Documentation Page

Overview

To build the server-side component of a “movies” web application.
The web application will provide users with access to information about different movies, directors, and genres.
Users will be able to sign up, update their personal information, and create a list of their favorite movies.

URL Endpoints

REQUEST URL HTTP METHOD REQUEST BODY DATA FORMAT RESPONSE BODY DATA FORMAT
Get a list of all movies /movies GET None A JSON object holding data about all the movies
Get information about a single movie by its title /movies/:title GET None A JSON object holding data about a single movie (title, year, genre, director):
{
title: 'Kung Fu Panda',
year: 2008,
genre: ['Animation', 'Action', 'Adventure'],
director: ['Mark Osborne','John Stevenson']
}
Get a list of all genres /genres GET None A JSON object holding data about all the genres
Get information about a genre by its name /genres/:genre GET None A JSON object holding data about a specific genre:
{
genreName: 'Action', description: 'An action story is similar to adventure, and the protagonist usually takes a risky turn, which leads to desperate situations'
}
Get a list of all directors /directors GET None A JSON object holding data about all directors
Get information about a director by name /directors/:name GET None A JSON object holding data about a specific genre:
{
name: 'Mark Osborne',
born: 'September 17, 1970'
}
Allow new users to register /register POST A JSON object holding data about the user to add, structured like:
{
username: 'username',
email: 'example@email.com'
} 		A JSON object holding data about the user that was added:
{
username: 'username',
email: 'example@email.com'
}
Get information about a user /users/:username GET None A JSON object holding data about the user:
{
username: 'username',
email: 'example@email.com'
}
Allow users to update their username /users/:username PUT None A text message indicating the username was updated.
Get a list of a user's favorite movies /users/:username/favorites GET None A JSON object holding data about a user's favorite movies.
Allow users to add a movie to their favorites /users/:username/favorites POST None A text message indicating the movie has been added to the user's favorites.
Allow users to remove a movie from their favorites /users/:username/favorites DELETE None A text message indicating the movie has been removed from the user's favorites.
Allow existing users to deregister /users/:username DELETE None A text message indicating the user has been deregistered.
